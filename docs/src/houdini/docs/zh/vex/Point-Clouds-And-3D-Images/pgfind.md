`int [] pgfind(<geometry>geometry, vector P, float radius, int maxpoints, float divsize)`

`int [] pgfind(<geometry>geometry, string ptgroup, vector P, float radius, int maxpoints, float divsize)`

## Arguments

`<geometry>`

当在一个节点的上下文中运行时（比如一个 wrangle SOP），这个参数可以是一个整数，代表要读取几何图形的输入数字（从 0 开始）。

或者，该参数可以是一个字符串，指定一个几何文件（例如，一个`.bgeo'）来读取。当在Houdini内部运行时，这可以是一个`op:/path/to/sop`的引用。

这些函数与`pcfind`函数非常相似。不同的是，它们使用基于网格的加速结构。这可以提供更快的初始化和查找，只要使用正确的网格大小调整参数。

如果你使用一个近乎恒定的搜索半径来搜索点云，那么这个半径就可以作为划分大小。

::: info Note

每个点的划分大小必须不一样。

::: info Note

分区大小被钳制在 3.0×105 以上。

`ptgroup`是一个点组，限制了搜索的点。这是一个 SOP 风格的组模式，所以可以是像`0-10`或`@Cd.x>0.5`。空白字符串被视为匹配所有点。
